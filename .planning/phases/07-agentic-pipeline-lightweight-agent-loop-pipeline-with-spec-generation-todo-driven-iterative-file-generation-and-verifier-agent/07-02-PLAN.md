<objective>
Implement the multi-turn file generation agent loop and the independent verifier LLM call — the two core mechanisms of the agentic pipeline.

Purpose: The file generator runs a multi-turn conversation where the LLM calls write_file/read_file tools to build up the game project iteratively. The verifier is a separate LLM call that audits all generated files and produces a structured error list. Together they form the generate-verify-fix loop.

Output: `file_generator.py` updated with `run_file_generation()`, new `verifier.py` with `run_verifier()`, and test coverage.
</objective>

    - `async def run_file_generation(client: AsyncAnthropic, spec: AgenticGameSpec, game_dir: Path, emit: EmitFn, *, context_strategy: str = "full_history") -> dict[str, str]`:
      1. Build initial user message from spec (include full spec JSON + instruction to start generating files)
      2. Initialize `messages` list and `generated_files` dict
      3. Loop up to MAX_TURNS_PER_ITERATION:
         a. Call `client.messages.create(model=GENERATOR_MODEL, max_tokens=8192, system=GENERATOR_SYSTEM_PROMPT, tools=AGENT_TOOLS, messages=messages)`
         b. Append assistant turn: `{"role": "assistant", "content": response.content}` — CRITICAL: append full content list, not just text
         c. If `response.stop_reason == "end_turn"`: break
         d. Process tool_use blocks: for each block with type=="tool_use", call `_dispatch_tool()`, collect tool_results
         e. If no tool_results: break (defensive)
         f. For each write_file dispatch, emit ProgressEvent with filename
         g. Append user turn: `{"role": "user", "content": tool_results}` — tool_result goes in USER role per Anthropic API
         h. In stateless mode: instead of accumulating, reset messages to a fresh prompt with `_build_stateless_prompt(spec, generated_files)` before step (a)
      4. Return generated_files dict

    - `def _build_stateless_prompt(spec: AgenticGameSpec, existing_files: dict[str, str]) -> str`: builds a prompt that lists the spec + existing file names (not contents — agent uses read_file for that).
