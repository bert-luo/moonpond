    - AgenticPipeline.generate() calls run_spec_generator, then enters generate-verify-fix loop, then calls run_exporter
    - Intermediate spec is saved as 1_agentic_spec.json in games/{game_dir}/intermediate/
    - Each iteration creates iteration_N/ subdir with files/ and verifier.json
    - Loop exits when verifier reports zero critical errors
    - Loop exits after MAX_ITERATIONS (3) even with remaining errors, proceeds to export with best available files
    - On fix iterations, only flagged files are passed to run_file_generation (targeted fixes per user decision)
    - Fix iteration system prompt includes original file content + verifier error context for each flagged file
    - Pipeline emits ProgressEvents for each major step (spec, iteration start, verifier result, export)
    - Pipeline handles exceptions by emitting error event + None sentinel (follow ContractPipeline pattern)
    - context_strategy parameter defaults to "full_history" (Claude's discretion per CONTEXT.md)
  </behavior>
  <action>
    Create `backend/backend/pipelines/agentic/pipeline.py` with:

    - Import `_slugify` pattern (copy the local helper from contract/pipeline.py — per project decision to keep _slugify local to each pipeline).
    - `MAX_ITERATIONS = 3` class constant (Claude's discretion per CONTEXT.md, suggested ~3).

    - `class AgenticPipeline:` with `__init__(self)` creating `self._client = AsyncAnthropic()`.

    - `async def generate(self, prompt: str, job_id: str, emit: EmitFn, *, save_intermediate: bool = True) -> GameResult:`:
      1. Emit stage_start "Starting agentic pipeline..."
      2. Call `run_spec_generator(self._client, prompt, emit)` -> spec
      3. Build game_dir slug: `f"{_slugify(spec.title)}_{timestamp}"` (same pattern as ContractPipeline)
      4. Create intermediate dir if save_intermediate. Save `1_agentic_spec.json`.
      5. Create game project dir at `GAMES_DIR / game_dir / "project"`.
      6. **Generate-Verify-Fix loop** (up to MAX_ITERATIONS):
         a. Emit "Iteration {n}: generating files..."
         b. On first iteration: call `run_file_generation(self._client, spec, project_dir, emit, context_strategy=self._context_strategy)` — generates all files
         c. On fix iterations: call `run_file_generation` with a modified prompt that includes only the flagged files + their verifier errors. Pass the fix context via a `fix_context` parameter (add this parameter to run_file_generation if not present — or build the fix prompt at pipeline level and pass as initial message override).
         d. Save iteration files to `intermediate/iteration_{n}/files/`
         e. Call `run_verifier(self._client, spec, all_files, emit)` -> verifier_result
         f. Save `intermediate/iteration_{n}/verifier.json`
         g. If not verifier_result.has_critical_errors: break
         h. Collect flagged files (file_path from critical errors) for next iteration
      7. Merge all generated files into final dict.
      8. Call `run_exporter(game_dir, all_files, [], emit)` -> result (controls=[] since agentic spec doesn't define controls in the same way).
      9. Emit done event with result data.
      10. Emit None sentinel.
      11. Return result.
      12. Wrap in try/except: emit error + None sentinel, re-raise (follow ContractPipeline pattern exactly).

    Add `_context_strategy: str = "full_history"` as instance attribute, settable via constructor parameter if desired.

    Use `patch` to mock the stage functions at the module level (e.g., `patch("backend.pipelines.agentic.pipeline.run_spec_generator")`). Use `tmp_path` for GAMES_DIR override.
