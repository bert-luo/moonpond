type Status = 'idle' | 'generating' | 'done' | 'error';

interface ChatMessage {
  id: string;
  type: 'stage' | 'complete' | 'error';
  text: string;

interface ControlMapping {
  key: string;
  action: string;
}

interface GenerationState {
  status: Status;
  messages: ChatMessage[];
  gameUrl: string | null;
  controls: ControlMapping[];
  errorMessage: string | null;

type GenerationAction =
  | { type: 'SUBMIT' }
  | { type: 'SSE_STAGE'; message: string }
  | { type: 'SSE_DONE'; gameUrl: string; controls: ControlMapping[] }
  | { type: 'SSE_ERROR'; message: string }
  | { type: 'RESET' };

function generationReducer(state: GenerationState, action: GenerationAction): GenerationState;
const initialState: GenerationState;

**File 1 — `frontend/hooks/useGeneration.ts`:**

Create a custom hook that encapsulates the fetch + EventSource lifecycle. It receives a `React.Dispatch<GenerationAction>` and returns `{ submit }`.

Implementation details:
- Use `useRef<EventSource | null>(null)` to track the active connection
- Use `useCallback` for `submit` with `dispatch` in the dependency array
- Use `NEXT_PUBLIC_BACKEND_URL` env var with fallback to `http://localhost:8000` for the backend base URL: `const BACKEND = process.env.NEXT_PUBLIC_BACKEND_URL ?? 'http://localhost:8000'`
- On `submit(prompt)`:
  1. Close previous EventSource if any: `esRef.current?.close()`
  2. Dispatch `{ type: 'SUBMIT' }`
  3. `fetch` POST to `${BACKEND}/api/generate` with JSON body `{ prompt }` and `Content-Type: application/json`
  4. Parse response JSON to get `job_id`
  5. If fetch fails (network error or non-ok response), dispatch `SSE_ERROR` with message and return early
  6. Create `new EventSource(${BACKEND}/api/stream/${job_id})`
  7. Store in `esRef.current`
  8. Register named event listeners:
     - `stage_start`: parse `e.data` as JSON, dispatch `SSE_STAGE` with `event.message`
     - `done`: parse `e.data` as JSON, extract `data.job_id` for gameUrl construction (`${BACKEND}/games/${data.job_id}/export/index.html`), extract `data.controls` (array of `{key, action}`), dispatch `SSE_DONE`, then `es.close()`
     - `error` (named event via addEventListener, NOT onerror): parse `(e as MessageEvent).data` as JSON, dispatch `SSE_ERROR` with `event.message`, then `es.close()`
  9. Register `es.onerror` for network failures: dispatch `SSE_ERROR` with "Connection lost", then `es.close()`
- `useEffect` cleanup on unmount: close `esRef.current`

CRITICAL: Register BOTH `es.addEventListener('error', handler)` for backend-emitted error events AND `es.onerror` for network failures. These are different — see research doc Pitfall 2.

**File 3 — `frontend/app/components/GameViewer.tsx`:**

"use client" component. Props:
```typescript
interface GameViewerProps {
  status: Status;
  gameUrl: string | null;
}
```

Renders one of three states:
1. **Idle** (`status === 'idle'` and no `gameUrl`): Centered placeholder text "Your game will appear here" with muted styling. Maybe a subtle game controller icon or emoji-free visual indicator.
2. **Generating** (`status === 'generating'`): Full-size loading skeleton with shimmer animation. Use the `shimmer` keyframe defined in globals.css (or `animate-pulse` as fallback). Show "Building your game..." text centered over the skeleton. The skeleton should feel alive per user decision — gradient sweep, not static gray.
3. **Done/Game loaded** (`gameUrl` is not null): An `<iframe>` filling the viewer:
   ```tsx
   <iframe
     src={gameUrl}
     className="w-full h-full border-0"
     sandbox="allow-scripts allow-same-origin"
     allow="cross-origin-isolated"
     title="Generated game"
   />
   ```
   CRITICAL: The `allow="cross-origin-isolated"` attribute is REQUIRED. Without it, the COOP/COEP headers set by the backend on `/games/*` will not propagate into the iframe context, and Godot WASM will fail to use SharedArrayBuffer. This is the iframe side of the cross-origin isolation chain (backend provides the headers, iframe must opt in via `allow`). See research doc Pattern 4 and Pitfall 1.

   The iframe should fill the entire viewer area. When transitioning from skeleton to iframe, the swap should be clean (no flash — conditionally render based on gameUrl).

If `status === 'error'` but `gameUrl` exists from a previous generation, keep showing the previous game iframe.
If `status === 'error'` and no `gameUrl`, show the idle placeholder.
