**Fix 2 — Add COOP/COEP middleware in main.py:**
Add a Starlette `BaseHTTPMiddleware` BEFORE the static file mount that injects cross-origin isolation headers on `/games/` responses. This is needed because the iframe embedding Godot WASM requires SharedArrayBuffer, which needs cross-origin isolation on BOTH the parent page (Next.js already has this) AND the embedded content.

Add this middleware class and registration:
```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

class COOPCOEPMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        response: Response = await call_next(request)
        if request.url.path.startswith("/games/"):
            response.headers["Cross-Origin-Opener-Policy"] = "same-origin"
            response.headers["Cross-Origin-Embedder-Policy"] = "require-corp"
            response.headers["Access-Control-Allow-Origin"] = "*"
        return response

app.add_middleware(COOPCOEPMiddleware)
```
Place the middleware addition AFTER CORSMiddleware addition (Starlette processes middleware in reverse order, so adding it after CORS means it runs before CORS).

Also update the default pipeline parameter in the generate endpoint from `pipeline: str = "stub"` to `pipeline: str = "multi_stage"` since the multi-stage pipeline is now the primary one.

**Step 3 — Create `frontend/app/globals.css`:**
Use Tailwind v4 `@import "tailwindcss"` syntax (NOT the old `@tailwind` directives). Define dark theme CSS variables using `oklch` color space for the Moonlake-inspired dark aesthetic:
- `--color-bg`: near-black with cool tint (oklch ~0.09)
- `--color-surface`: slightly lighter for panels/cards (oklch ~0.13)
- `--color-border`: subtle borders (oklch ~0.22)
- `--color-text`: high-contrast primary text (oklch ~0.93)
- `--color-text-muted`: secondary/dimmed text (oklch ~0.55)
- `--color-accent`: indigo/blue accent for buttons and highlights (oklch ~0.72 with chroma ~0.18)
- `--color-glow`: semi-transparent accent for glow effects

Register theme colors with `@theme inline` block mapping to the CSS variables.

Add a `@keyframes shimmer` animation for the loading skeleton (gradient sweep left to right over 1.5s infinite).

**Step 4 — Create `frontend/types/generation.ts`:**
Define all types for the generation flow:
- `Status` = `'idle' | 'generating' | 'done' | 'error'`
- `ChatMessage` = `{ id: string; type: 'stage' | 'complete' | 'error'; text: string }`
- `ControlMapping` = `{ key: string; action: string }`
- `GenerationState` = `{ status: Status; messages: ChatMessage[]; gameUrl: string | null; controls: ControlMapping[]; errorMessage: string | null }`
- `GenerationAction` discriminated union with 5 cases: `SUBMIT`, `SSE_STAGE`, `SSE_DONE`, `SSE_ERROR`, `RESET`
- `SSEProgressEvent` = `{ type: 'stage_start' | 'done' | 'error'; message: string; data: Record<string, unknown> }`
- `generationReducer` function implementing all state transitions per the research doc pattern
- `initialState` constant

For `SSE_DONE`: set status to `'done'`, store `gameUrl` and `controls`, append a "complete" message.
For `RESET`: set status back to `'idle'` but keep `gameUrl` visible (game stays in iframe).
For `SUBMIT`: clear messages, gameUrl, controls, errorMessage; set status to `'generating'`.

Use `crypto.randomUUID()` for message IDs.
