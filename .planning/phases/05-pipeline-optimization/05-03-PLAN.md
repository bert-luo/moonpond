**Public function:**
```python
async def run_parallel_node_generation(
    client: AsyncAnthropic,
    contract: GameContract,
    emit: EmitFn,
) -> dict[str, str]:
    """Generate all node scripts in parallel (leaves) then sequential (orchestrators).
    Returns dict mapping filename -> file content for all .gd and .tscn files."""
```

**Implementation:**
1. Separate nodes into leaves (empty `dependencies` list) and orchestrators (non-empty `dependencies`)
2. Emit ProgressEvent(type="stage_start", message=f"Generating {len(leaves)} game files in parallel...")
3. Use `asyncio.gather(*[_generate_single_node(client, node, contract) for node in leaves], return_exceptions=True)`
4. Process results: collect successful dict[str, str] results, log failed nodes
5. If any failed, emit ProgressEvent warning with count
6. Generate orchestrator nodes sequentially (they depend on leaf contracts being defined, though not on leaf code)
7. Merge all files into a single dict and return

**Private function:**
```python
async def _generate_single_node(
    client: AsyncAnthropic,
    node: NodeContract,
    contract: GameContract,
) -> dict[str, str]:
    """Generate .gd (and optionally .tscn) for a single node."""
```

Implementation:
1. Build system prompt:
   - "You are generating GDScript for a Godot 4 game. Generate ONLY the file(s) for this node."
   - "Do NOT generate game_manager.gd — GameManager is a pre-existing autoload."
   - "Respond with ONLY files for: {node.script_path}" (and scene_path if applicable)
   - "Use Godot 4 syntax exclusively. Use Input.is_action_pressed() with these actions: {INPUT_ACTIONS}"
   - Include available shaders, palettes, particles from models.py constants
   - Include the full GameContract JSON for cross-reference
   - "Implement EXACTLY these methods: {node.methods}. Do not rename or add methods not in the contract."
   - "Emit EXACTLY these signals: {node.signals}"
   - "Add to EXACTLY these groups: {node.groups}"
2. Build user message with the specific NodeContract JSON
3. Call `client.messages.create()` with model="claude-sonnet-4-20250514", max_tokens=8192
4. Parse the JSON response: expect `{"script_path": "code content", ...}`
5. Return the dict

Use `json.loads()` to parse. If the response contains markdown code fences, strip them before parsing.

Create `backend/backend/stages/wiring_generator.py`:

```python
async def run_wiring_generator(
    client: AsyncAnthropic,
    contract: GameContract,
    generated_files: dict[str, str],
    emit: EmitFn,
) -> dict[str, str]:
    """Generate Main.tscn and optionally project.godot from the contract and generated files.
    Returns dict mapping filename -> content for wiring files only."""
```

**Implementation:**
1. Emit ProgressEvent(type="stage_start", message="Wiring scene tree...")
2. Build a system prompt for Main.tscn generation:
   - "Generate a Godot 4 text-format Main.tscn file that wires all these nodes together."
   - Include the full GameContract JSON
   - Include the list of actually-generated files (keys of generated_files dict)
   - "Use [ext_resource] with unique incrementing integer IDs for each script."
   - "Each node in the scene tree must reference the correct script via its ext_resource ID."
   - "The root node should be a Node2D named 'Main'."
   - "Connect signals as specified in the node contracts."
3. Call LLM with model="claude-sonnet-4-20250514", max_tokens=4096
4. Parse the response to extract Main.tscn content
5. Add "Main.tscn" to the output dict
6. If `contract.autoloads` is non-empty:
   - Read the template project.godot from TEMPLATE_DIR
   - Use regex to replace only the `[autoload]` section
   - Keep `[input]` section untouched (CRITICAL — this preserves the 8 standard input actions)
   - Add "project.godot" to the output dict
7. Return the wiring files dict

**CRITICAL anti-patterns to avoid (per RESEARCH.md):**
- Do NOT generate project.godot from scratch — it would lose input action mappings
- Do NOT let node generators write Main.tscn — only the wiring stage does this
- Use incrementing integer IDs for ext_resources to prevent collisions

Tests: Create a mock contract with 3 nodes, mock the LLM to return a valid Main.tscn, verify the output. Test the project.godot patching path by providing a mock template content and non-empty autoloads.
