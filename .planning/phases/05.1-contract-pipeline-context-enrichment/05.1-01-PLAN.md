Purpose: Nodes currently hallucinate GameManager APIs because the static template game_manager.gd only has palette/state helpers. Generating a game-specific game_manager.gd from the contract ensures the runtime API matches what nodes expect.

Output: New `game_manager_generator.py` module, updated `contract_models.py` with new fields, updated pipeline wiring, tests.
