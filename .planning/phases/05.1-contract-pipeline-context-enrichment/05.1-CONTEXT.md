# Phase 05.1: Contract Pipeline Context Enrichment - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning
**Source:** Root cause analysis of broken cookie-kingdom-clicker game output

<domain>
## Phase Boundary

This phase fixes the core quality gap in the contract pipeline: nodes are generated in isolation and hallucinate GameManager APIs that don't exist, resulting in broken games. Three targeted improvements to the node generation context solve this without changing the pipeline architecture.

Scope: backend/backend/stages/ only. No frontend, no API changes, no new pipeline stages.

</domain>

<decisions>
## Implementation Decisions

### 1. GameManager Generator (CTXE-01)
- Generate game_manager.gd from GameContract fields (game_manager_enums, game_manager_properties, and methods inferred from node contracts)
- This runs AFTER contract generation but BEFORE node generation (new Stage 2.5 or folded into wiring generator)
- Generated game_manager.gd replaces the static template copy during export
- Must preserve the existing palette/state API from the template (set_palette, get_palette_color, set_state, GameState enum)
- Must ADD game-specific properties, methods, signals, and enums from the contract
- The contract model should gain a `game_manager_methods` field so the contract generator can explicitly declare what GameManager exposes

### 2. GameManager API injection into node prompts (CTXE-02)
- In `_build_node_system_prompt()` (node_generator.py), add a structured block:
  ```
  GameManager API (autoload — available globally):
    Properties: currency, income_per_second, ...
    Methods: add_currency(amount: int), can_afford(cost: int) -> bool, ...
    Signals: currency_changed, income_changed, ...
    Enums: GameState { PLAYING, PAUSED }, UpgradeType { CLICK_VALUE, AUTO_CLICKER, MULTIPLIER }
  ```
- Derived from contract.game_manager_properties, contract.game_manager_enums, and the new game_manager_methods field
- Replace the vague "GameManager is a pre-existing autoload" with this concrete API spec

### 3. Sibling dependency API injection (CTXE-03)
- In `_build_node_system_prompt()`, for each entry in node.dependencies, look up that NodeContract and inject:
  ```
  Dependency: player.gd (Node)
    Methods: add_currency(amount: int), can_afford(cost: int) -> bool
    Signals: currency_changed, income_changed
    Groups: game_managers
  ```
- This replaces dumping the entire contract JSON (which the LLM must parse) with focused, relevant API blocks
- Keep the full contract JSON dump as well for broader context, but the focused blocks take priority

### Claude's Discretion
- Whether to add game_manager_methods as a new field on GameContract or infer methods from node contracts
- Whether the GameManager generator is a separate stage function or integrated into an existing stage
- Exact prompt formatting for the API blocks
- How to handle the template game_manager.gd base (extend it vs replace it entirely)

</decisions>

<specifics>
## Specific Ideas

### Root cause evidence (from cookie-kingdom-clicker analysis)
- click_area.gd called GameManager.add_currency() — doesn't exist on template
- player.gd called GameManager.pause_game() — doesn't exist
- upgrade_panel.gd referenced GameManager.currency_changed signal — doesn't exist
- upgrade_button.gd called GameManager.get_currency()/can_afford()/purchase_upgrade() — none exist
- The contract DID specify game_manager_properties: [currency, income_per_second, total_clicks, upgrades_purchased]
- The contract DID specify game_manager_enums: {GameState: [PLAYING, PAUSED, MENU], UpgradeType: [CLICK_VALUE, AUTO_CLICKER, MULTIPLIER]}
- But nothing consumed these to generate the actual game_manager.gd or tell nodes what was available

### Existing code locations
- Node generation prompt: backend/backend/stages/node_generator.py `_build_node_system_prompt()`
- Contract model: backend/backend/stages/contract_models.py GameContract class
- Contract generator prompt: backend/backend/stages/contract_generator.py `_CONTRACT_GENERATOR_SYSTEM_PROMPT`
- Pipeline wiring: backend/backend/pipelines/contract/pipeline.py `generate()`
- Template game_manager.gd: godot/templates/base_2d/game_manager.gd

</specifics>
