<objective>
Eliminate invalid ExtResource IDs (Bug C) and wrong-node script attachment (Bug F) by stripping per-node .tscn files from the pipeline. The wiring stage already owns Main.tscn assembly from the contract — per-node .tscn files are redundant and error-prone.

Purpose: Per-node .tscn files generated by the LLM frequently contain mismatched ExtResource IDs and incorrect script attachments. Rather than validating/fixing these files, the simpler and more reliable approach is to strip them entirely and let the wiring stage (which already has the full contract) handle all scene assembly.
Output: Patched pipeline.py that filters .tscn from node output, with tests confirming the behavior.
</objective>
