<task type="auto">
  <name>Task 2: Implement Prompt Enhancer and Game Designer stages</name>
  <files>backend/backend/stages/prompt_enhancer.py, backend/backend/stages/game_designer.py</files>
  <action>
1. Create `backend/backend/stages/prompt_enhancer.py`:

- Define `HAIKU_MODEL = "claude-haiku-4-5"`
- Define a system prompt that instructs the LLM to analyze a game idea and return a structured JSON object with fields: title, genre, mechanics (list), visual_hints (list). The system prompt should tell the LLM to respond ONLY with valid JSON matching the GameSpec schema.
- Implement `async def run_prompt_enhancer(client: AsyncAnthropic, prompt: str, emit: EmitFn) -> GameSpec`:
  - Emit `ProgressEvent(type="stage_start", message="Understanding your idea...")`
  - Call `await client.messages.create()` with model=HAIKU_MODEL, max_tokens=512, the system prompt, and user message = prompt
  - Parse the response text as JSON and validate with `GameSpec.model_validate()`
  - Return the validated GameSpec
- NOTE: Use `client.messages.create()` + `json.loads()` + `GameSpec.model_validate()` rather than `messages.parse()`. The research notes that `messages.parse()` async availability is uncertain (Open Question #2). The fallback pattern is safer.

2. Create `backend/backend/stages/game_designer.py`:

- Define `SONNET_MODEL = "claude-sonnet-4-6"`
- Define a system prompt that instructs the LLM to take a GameSpec and produce a full GameDesign. The prompt must:
  - Explain the ControlScheme enum values and when to use each
  - Explain the ControlMapping format (key = human-readable label, action = description)
  - Explain SceneSpec format (name, description, nodes as Godot node type names)
  - Explain VisualStyle (palette must be one of: neon/retro/pastel/monochrome; shader must be one of: pixel_art/glow/scanlines/chromatic_aberration/screen_distortion)
  - Tell the LLM to respond ONLY with valid JSON matching the GameDesign schema
- Implement `async def run_game_designer(client: AsyncAnthropic, game_spec: GameSpec, emit: EmitFn) -> GameDesign`:
  - Emit `ProgressEvent(type="stage_start", message="Designing game structure...")`
  - Call `await client.messages.create()` with model=SONNET_MODEL, max_tokens=2048, the system prompt, and user message = `game_spec.model_dump_json()`
  - Parse the response text as JSON and validate with `GameDesign.model_validate()`
  - Return the validated GameDesign

Both stage functions take `client: AsyncAnthropic` as first argument (shared client pattern from research). Both use `json.loads()` to parse the text response, stripping any markdown code fences the LLM might include before parsing.
