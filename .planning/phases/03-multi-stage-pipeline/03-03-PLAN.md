2. Create `backend/backend/pipelines/multi_stage/pipeline.py`:

```python
from anthropic import AsyncAnthropic

MAX_RETRIES = 2
```

- Implement `class MultiStagePipeline`:
  - `__init__(self)`: Create `self._client = AsyncAnthropic()` (reads ANTHROPIC_API_KEY from env). Do NOT create at module level (Pitfall 1).
  - `async def generate(self, prompt: str, job_id: str, emit: EmitFn) -> GameResult`:
    - Stage 1: `game_spec = await run_prompt_enhancer(self._client, prompt, emit)`
    - Stage 2: `game_design = await run_game_designer(self._client, game_spec, emit)`
    - Stage 3 with self-correction: Call `_generate_code_with_correction(self._client, game_design, emit)` — see below
    - Stage 4: `polished_files = await run_visual_polisher(self._client, files, game_design.visual_style, emit)`
    - Stage 5: `result = await run_exporter(job_id, polished_files, [c.model_dump() for c in game_design.controls], emit)`
    - Emit `ProgressEvent(type="done", message="Your game is ready.")`
    - Emit `None` to signal stream end
    - Return result
  - Wrap the entire generate body in try/except. On exception: emit `ProgressEvent(type="error", message=str(e))`, emit `None`, then re-raise.

- Implement `async def _generate_code_with_correction(client, game_design, emit) -> dict[str, str]` as a module-level helper:
  - First call: `files = await run_code_generator(client, game_design, emit, previous_error=None)`
  - Check: `syntax_error = _check_gdscript_syntax_patterns(files)`
  - If no error: return files
  - Loop up to MAX_RETRIES times:
    - `files = await run_code_generator(client, game_design, emit, previous_error=syntax_error)`
    - Re-check syntax
    - If clean: return files
  - After MAX_RETRIES exhausted: raise `RuntimeError(f"Code generation failed after {MAX_RETRIES} retries: {syntax_error}")`
